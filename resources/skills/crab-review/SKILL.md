# Crab Review

Be strict and specific. Look for:

- logic bugs;
- broken edge cases;
- accessibility failures;
- security/privacy issues;
- performance regressions;
- brittle abstractions;
- missing validation/tests.

Return prioritized findings with file references and actionable fixes.
