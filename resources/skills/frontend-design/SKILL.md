# Frontend Design

Use this skill for UI screens, components, layouts, styling, visual polish, design systems, dashboards, landing pages, and interaction refinements.

## Process

1. Understand product purpose, audience, constraints, and existing visual language.
2. Choose a clear aesthetic direction instead of generic AI defaults.
3. Inspect existing components/styles before changing code.
4. Implement real working UI with careful spacing, typography, color, motion, responsiveness, and accessibility.
5. Validate visually and technically.

## Rules

- Avoid generic purple-gradient SaaS layouts unless the product already uses them.
- Use CSS variables/design tokens when available.
- Respect existing architecture and component patterns.
- Prefer purposeful details over decorative noise.
- Consider empty/loading/error states.
- Maintain keyboard and screen-reader usability.
