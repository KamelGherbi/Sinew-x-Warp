# Clean Code

Improve code without changing behavior.

## Checklist

- Read the surrounding code first.
- Keep changes local and reversible.
- Prefer clear names over comments.
- Remove duplication only when the abstraction is obvious.
- Simplify control flow and early returns.
- Preserve public APIs unless explicitly requested.
- Run targeted checks after edits.

Do not perform broad refactors or aesthetic rewrites unless the user asks for them.
