# Review Code

Perform an adversarial but constructive review.

## Review areas

- Correctness and edge cases.
- Data loss, race conditions, async ordering.
- Error handling and recovery.
- UX consistency and accessibility.
- Security/privacy risks.
- Performance and unnecessary work.
- Test and validation gaps.

Return findings prioritized by severity with concrete file references and suggested fixes.
