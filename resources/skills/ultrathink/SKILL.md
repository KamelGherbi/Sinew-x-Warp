# Ultrathink

Use for hard problems where rushing is risky.

## Approach

- Build context before concluding.
- Separate facts, assumptions, and unknowns.
- Consider multiple plausible causes or designs.
- Choose one path and explain why.
- Identify failure modes and validation steps.
- Keep final output concise and actionable.
