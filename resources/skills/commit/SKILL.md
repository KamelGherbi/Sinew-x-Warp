# Commit

Use when the user wants to save work in Git.

## Steps

1. Inspect `git status` and relevant diff.
2. Ensure only intended files are included.
3. Run appropriate checks if not already done.
4. Stage selected files.
5. Commit with a concise conventional message.
6. Report commit hash.

Never push unless explicitly asked.
