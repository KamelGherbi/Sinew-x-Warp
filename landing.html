Ch. 03 — Three modes

One agent. Three temperaments. You pick the autonomy. It stays in lane.

01

Act.

Hands on the keyboard. Reads, writes, runs, fixes.

Every file change shows you a diff before it lands. The default when you know where you're going and just want momentum.

Default · Move fast
02

Plan.

Cannot touch a single file. Asks until you tell it to stop.

For risky migrations and unknown codebases. The agent investigates, asks one question at a time, then writes a plan you can run in one click.

Read-only · Ask · Plan
03

Goal.

Set the target. Walk away.

Turn after turn, the agent keeps going — until it can prove the goal is done. Pause anytime. Resume tomorrow.

Autonomous · Pausable

Spin up a swarm of agents, each with its own model and personality. They share one board and one channel. They figure it out together.

No middle manager. No status meetings. Everyone sees what everyone else is doing in real time.

Ch. 06 — Extend

The base agent is just the start. Teach it your domain. Plug in your tools. Spin up specialists. It learns what you teach it.

A Loadable

Skills.

Markdown files of expertise. The agent loads them on demand, not all the time.

Drop a skill into your repo (visible to teammates) or your home folder (yours). Front-end design, refactor playbooks, your team's conventions — whatever you want the agent to know when it needs to.

B Bridge

MCP.

Plug any Model Context Protocol server. Your tools become the agent's hands.

Add servers in Settings, probe them to make sure they're alive, and the agent picks them up. Tools load lazily — the agent only sees the schemas it actually needs.

C Reusable

Sub-agents.

Persistent specialists. Each with its own model, prompt, and lane.

Save a code-reviewer with a Sonnet + your style guide. Save a researcher with a high-thinking model. The main agent delegates to them by name and they report back.
